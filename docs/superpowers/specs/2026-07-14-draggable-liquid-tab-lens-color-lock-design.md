# 可拖动液态玻璃 Tab 镜片与颜色锁定设计

日期：2026-07-14

## 1. 目标

在现有 SwiftUI 自定义底部导航栏上增加点击横移、水平跟手拖动、快速甩动、轻微液态拉伸、弹簧吸附和内容放大，同时将活动液态玻璃镜片的基础颜色设为最高优先级不变量。

镜片在任何 Tab、位置、速度、形状和动画阶段都必须使用修改前完全相同的设计 Token、Tint、材质、透明度和饱和度。背景内容经过镜片时允许产生原生折射和高光变化，但不得通过应用状态改变镜片基础色。

## 2. 当前视觉基线

当前项目的品牌色来源为 `Assets.xcassets/AccentColor.colorset`：

- sRGB red：`0.880`
- sRGB green：`0.200`
- sRGB blue：`0.320`
- alpha：`1.000`

当前活动镜片参数：

- iOS 26 Glass：`.regular.tint(lensTint.opacity(0.14)).interactive()`
- Glass opacity：`1.0`
- Saturation：`1.0`
- Highlight opacity：`0.52`
- Highlight line width：`0.8`
- Shadow tint opacity：`0.10`
- Shadow radius：`6`
- Shadow y：`2`
- iOS 17–25 fallback：`.ultraThinMaterial` 加相同 `0.14` Tint、描边和阴影

实施前必须在模拟器保存静止镜片基线截图。上述参数和最终静止视觉不得改变。

## 3. 颜色与内容分层

### 3.1 镜片层

`ActiveTabGlassStyle` 只渲染玻璃镜片，不再承载 Tab 图标。它直接引用现有 `Color.accentColor` 设计 Token，不复制 RGB，也不从当前 Tab、拖动目标或图标颜色派生 Tint。

镜片样式输入只允许包含：

- 固定 Lens Tint Token
- 固定 Tint opacity
- 固定 Glass opacity
- 固定 Saturation
- 固定 Highlight 参数
- 动态几何形状

动态几何只包括中心位置、宽度、高度和圆角。禁止向样式传入 Tab、选中状态、拖动进度、速度或动画进度。

### 3.2 内容层

所有 Tab 图标保留在独立内容层。图标颜色和缩放可以根据交互状态变化，但不得修改、覆盖或向下传递到镜片层。

- 静止选中图标继续使用现有品牌色。
- 未选中图标继续使用现有主色透明度。
- 拖动时离镜片中心最近的图标按距离平滑放大，最大为 `1.08`。
- 拖动经过某个图标时，可以在内容层临时显示选中图标色；页面选择只在拖动结束后提交。
- 点击 Tab 时沿用当前导航语义：立即提交页面选择，同时让镜片弹簧移动到目标槽位。

## 4. 单一持续镜片

导航栏始终只渲染一个活动镜片 View 和一个 `ActiveTabGlassStyle`。点击、拖动、甩动和回弹都只改变该 View 的几何状态，不创建起点镜片、终点镜片或中间镜片，也不通过 opacity 交叉淡入淡出。

不引入第二个 Tint、渐变、品牌色覆盖、brightness、hue rotation 或 saturation 动画。现有外围玻璃胶囊和活动镜片的材质组合保持不变，避免因更换容器或采样方式改变已确认的静止颜色。

## 5. 交互模型

新增纯值类型 `TabBarLensInteractionModel`，负责把导航栏宽度、当前选择、拖动位移和预测终点转换为可测试的几何状态。

模型输出：

- `centerX`：镜片当前中心，限制在首尾 Tab 中心之间。
- `width`：静止宽度加轻微水平拉伸，拉伸增量最大 `10pt`。
- `height`：保持当前 `54pt`。
- `cornerRadius`：保持当前 `27pt`，确保终点静止形状不变。
- `hoveredTab`：当前离镜片中心最近的 Tab。
- `magnification(for:)`：按图标中心与镜片中心距离映射到 `1.0...1.08`。
- `targetTab`：根据拖动预测终点选择最近 Tab。

拖动开始时记录当前选中 Tab 的槽位中心。拖动过程中使用手势 translation 计算中心位置；拉伸量从当前移动速度和位移方向派生。为保持物理连续性，镜片中心需补偿拉伸锚点，使拉伸主要出现在移动方向前端，而不是两侧平均膨胀。

拖动结束时使用 `predictedEndTranslation.width` 计算甩动目标，选择预测中心最近的 Tab，并只提交一次 `selection`。随后使用统一弹簧吸附：

```swift
.spring(
    response: 0.36,
    dampingFraction: 0.88,
    blendDuration: 0.08
)
```

快速连续点击或新手势会从当前呈现几何继续，不等待上一段动画完成。

## 6. 收起状态与滚动状态

- 横向拖动只在完整导航栏展开时生效。
- 收起后的圆形镜片继续只执行“展开导航栏”，不在收起状态切换 Tab。
- 当前 20pt 收起阈值、16pt 展开阈值和 0.40 秒状态锁保持不变。
- 手势只附着在底部导航栏区域，不改变页面 ScrollView 的滚动数据流；如果外部状态切换为收起，立即结束临时拖动状态并沿用现有收起动画。
- 收起/展开动画继续使用同一个镜片和同一个固定样式。

## 7. Reduce Motion 与辅助功能

- 开启 Reduce Motion 时仍允许直接跟手拖动。
- 结束时使用短 `easeOut` 吸附，不使用速度拉伸、回弹或图标放大。
- 五个 Tab 按钮继续保留各自标签、标识和点击行为。
- 拖动手势不能移除或遮挡按钮的辅助功能命中区域。
- 收起状态继续暴露唯一 `collapsed-tab-<tab>` 控件。

## 8. 性能与视觉约束

- 不使用自定义 Shader、离屏快照镜片或额外 blur/material 叠层。
- 不对整个导航栏执行 scaleEffect 或 clipShape。
- 镜片只更新少量标量几何值；图标放大由距离直接计算。
- 保持一个几何弹簧，禁止 position、width、cornerRadius 使用相互冲突的曲线。
- 若原生互动高光在快速移动时造成明显发白或发灰，优先降低动态高光响应，不改变固定 Tint、Tint opacity、Glass opacity 或 Saturation。

## 9. 测试策略

### 9.1 单元测试

先写失败测试，再实现：

1. 所有 `AppTab`、收起进度、拖动位移、拉伸量和速度返回相同 Lens Token 与固定外观标量。
2. Lens 外观结构不接受 Tab、拖动进度或速度作为输入。
3. 拖动中心限制在首尾槽位之间。
4. 慢拖选择最近 Tab，快速甩动使用预测终点选择目标。
5. 拉伸不超过 `10pt`，静止时为零。
6. 图标放大限制在 `1.0...1.08`，只影响内容层。
7. 点击选择、拖动目标和收起选择继续使用同一个 `AppTab` 数据源。

### 9.2 UI 测试

1. 依次点击五个 Tab，确认只有一个活动镜片，页面导航正确。
2. 从第一个 Tab 慢拖到最后一个 Tab，确认目标正确且按钮保持可访问。
3. 快速甩动跨越多个 Tab，确认只提交一个最终选择。
4. 快速连续点击多个 Tab，确认镜片不复制、不闪烁。
5. 在收起状态尝试横向移动，确认只保留展开行为。
6. 保留现有多攒钱目标、快速上下滚动和立即反向测试。

### 9.3 模拟器颜色验收

在同一 iPhone 模拟器、相同外观设置下：

1. 修改前记录首个 Tab 静止镜片截图。
2. 修改后分别记录五个 Tab 静止镜片截图。
3. 以 60fps 录制慢拖、快速甩动、拉伸和回弹。
4. 抽取中间帧，检查是否出现变红、变白、变灰、变深、变浅、Tint 闪烁或双玻璃叠加。
5. 代码级验收以相同 Token 身份和固定外观标量为准；像素级只用于检查异常闪变，因为原生折射会随背景内容产生允许的局部差异。
6. 检查运行日志，不得出现崩溃、Swift 运行时错误或布局约束冲突。

## 10. 不在本次范围

- 不修改页面内容、数据模型、导航目标、Tab 顺序或图标资源。
- 不修改外围胶囊的静止颜色和材质。
- 不改变收起后的尺寸、位置和点击展开逻辑。
- 不加入每个 Tab 独立镜片颜色。
- 不使用拖动实时切换页面内容。
- 不采用自定义 Metal Shader 或高成本实时截图放大。

## 11. 验收标准

- 镜片在所有五个 Tab 使用同一个现有 AccentColor Token 和完全相同的固定外观参数。
- 点击、慢拖、甩动、拉伸、吸附、回弹和快速连续切换均不修改镜片 Tint、opacity、saturation 或材质。
- 镜片与图标内容层完全解耦；图标颜色变化不影响镜片。
- 拖动跟手，甩动目标正确，轻微拉伸不超过 `10pt`，吸附无明显弹跳。
- 镜片经过图标时只有几何、折射、高光和内容放大发生变化。
- 始终只有一个活动镜片，无复制、重影、颜色交叉淡入或第二层玻璃。
- 收起/展开、滚动锁、内容避让和业务导航保持现有行为。
- 相关单元测试、UI 测试、构建、模拟器视觉检查和日志检查全部通过。
