# 发给 Codex 的提示词

下面不要一次全部发送。按阶段逐段复制，效果更稳。

## 第一阶段：只检查，不允许改代码

请先阅读仓库根目录的 `AGENTS.md`、`DESIGN_TOKENS.json`、`ASSET_MAP.md`、`SCREEN_CHECKLIST.md`，并检查 `DesignReferences/NewUI` 与 `DesignReferences/OriginalUI` 中的界面图片。

目标是把现有记账 App 的视觉层改成 `DesignReferences/NewUI` 的弗瑞品牌风格，但必须完整保留当前页面主体框架、业务逻辑、数据结构、导航关系、交互能力、持久化与同步机制。

本轮严格禁止修改任何代码。请先完成项目审计并输出：

1. 当前使用的 UI 技术栈和最低系统版本。
2. 五个页面分别对应哪些 View、ViewModel、Model 和路由文件。
3. 当前底部 TabBar、卡片、标题、列表行、图表和日历是否已有共享组件。
4. 哪些文件属于纯视觉层，可以安全修改。
5. 哪些文件包含业务逻辑，本次不得修改。
6. 建议新增的共享主题与组件文件。
7. 分阶段实施顺序，每个阶段预计修改哪些文件。
8. 你发现的风险或效果图与真实功能之间的冲突。

请等我确认计划后再开始编码。

## 第二阶段：建立设计系统，不改具体页面结构

根据已确认的审计结果，先建立共享视觉系统，但暂时不要重写五个页面的主体布局。

要求：

- 使用当前项目原有 UI 框架，不迁移技术栈。
- 根据 `DESIGN_TOKENS.json` 建立颜色、字号、间距、圆角、阴影和渐变的统一定义。
- 新增或整理可复用组件，例如 BrandedCard、BrandedSectionHeader、BrandedIconTile、MascotDecoration、品牌化按钮和底部导航样式适配器。
- 不改 ViewModel、Model、数据读取、金额计算、导航目标、持久化、同步或导出逻辑。
- 不引入第三方依赖。
- 保留 Dynamic Type、VoiceOver、Reduce Motion 和安全区支持。
- 角色装饰必须 `allowsHitTesting(false)`，不能遮挡交互。

完成后：

1. 运行构建和现有测试。
2. 列出所有修改文件。
3. 说明每个共享组件的用途。
4. 不要继续改具体页面，等待我确认。

## 第四阶段：五个页面的具体指令

### A. 账单页

请改造账单页，对照 `DesignReferences/NewUI/01_Bill_账单.png`。

- 保留月份、搜索、筛选、收入/支出/结余、日期分组、交易分类、账户来源和所有点击行为。
- 顶部使用品牌标题、轻量授设趴边装饰、三张财务汇总卡。
- 交易列表改为更清晰的圆角卡片，但仍使用真实交易数据。
- 授设小贴纸只在空间充足时作为右上角装饰，不能影响复用和滚动性能。
- 当前 Tab 必须正确高亮。

### B. 日常账本页

请改造日常账本页，对照 `DesignReferences/NewUI/02_Ledger_日常账本.png`。

- 保留月历计算、月份切换、选中日期、每日收支显示和下方明细联动。
- 保留小票、资产、清单、预算四个入口及其原路由。
- 使用品牌化日历卡、选中描边和空状态角色插画。
- 当前 Tab 必须是“账本”高亮，不能误高亮“账单”。

### C. 攒钱页

请改造攒钱页，对照 `DesignReferences/NewUI/03_Savings_攒钱.png`。

- 保留目标、已存、目标金额、剩余天数、自动计划、存入操作和识别状态。
- 目标卡片可以根据目标类型显示旅行/家居类插画，但数据来自现有模型。
- 进度条和百分比必须由真实数据计算，避免重复计算逻辑。
- 当前 Tab 必须正确高亮。

### D. 明细页

请改造明细页，对照 `DesignReferences/NewUI/04_Analytics_明细.png`。

- 保留个人/家庭切换、月份切换、支出/收入/预算/趋势等筛选。
- 保留原图表数据源和交互。
- 图表卡改为品牌风格，中心显示总额，下方保留分类、占比和金额。
- 颜色既符合品牌，也必须保持财务语义和可访问性。

### E. 更多页

请改造更多页，对照 `DesignReferences/NewUI/05_More_更多.png`。

- 保留所有原设置项、路由和状态。
- 家庭账户卡和头像区域只做视觉改造。
- 设置列表使用统一卡片、图标底板和分区标题。
- 效果图中没展示但代码中存在的设置项，放入最合理的现有分组，不得删除。

## 第五阶段：视觉回归与收尾

请对五个已改页面进行最终检查，不新增功能，不做无关重构。

5. 小屏、大字体、深色模式策略、VoiceOver 和 Reduce Motion 的基本情况。
6. 构建、测试、lint 的结果。
7. 本次修改的完整文件列表。
8. 仍需由我提供的正式美术素材清单。

如能运行模拟器，请为五个页面各生成一张最新截图，并与 `DesignReferences/NewUI` 对照说明主要差异。不要为了追求像素级一致而破坏原功能或可访问性。
